# Contribuindo

Obrigado por ajudar no projeto ClaudiaPalooza. Siga estas regras básicas:

- Use branches por feature: `git checkout -b feat/descricao`
- Faça commits pequenos e com mensagens claras.
- Rode os scripts localmente em `--dry-run` antes de abrir PRs que alterem escrita em planilhas.
- Não inclua `credentials/service_account.json` em commits.
- Documente mudanças significativas no `README.md`.

Pull request checklist:

- [ ] Descreve a mudança e porquê
- [ ] Testado localmente (dry-run quando pertinente)
- [ ] Atualiza documentação se necessário
